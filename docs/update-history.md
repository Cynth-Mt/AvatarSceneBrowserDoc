# 更新日志

## [0.02] - 2025.6.28

- 新增路径与GUID双重错误检查，支持路径和GUID互相修复，场景丢失问题大幅减少
- 启动和加载速度提升，刷新时有进度条提示
- 优化TagManagementWindow的UI布局

## [0.03] - 2025.7.19

- 截图排序功能，现在是依照截图的时间来排序

## [0.04] - 2025.8.23

- 双击直接打开场景
- 增强右键菜单，限定在缩略图上右键，无需先选中
- 新增文件定位：Unity项目窗口显示、系统文件浏览器显示
- 解决新截图无法实时显示的问题
- 修复GUI刷新延迟和错误
- 根据标签背景色自动选择黑/白文字
- 截图格式/质量设置：支持 PNG/JPEG 切换，保存时自动清理同名旧文件
  - 新增 `Screenshot Settings...` 设置窗口
  - 菜单统一：所有入口归于 `Tools/CYN-lab/AvatarSceneBrowser/`
- 引入事件总线，改善多个窗口之间的通信

## [0.1] - 2025.10
BOOTH 正式发布

## [0.11] - 2025.11.14
添加操作说明文档

## [0.2] - 2026.1

**一个场景内多个 VRC Avatar 的管理功能**

- 支持在同一场景中为不同的 Avatar 分别截图并保存配置
- 点击配置缩略图可以一键恢复对应 Avatar 的显示/隐藏状态
- 画廊中绿色边框为 Avatar 配置截图，蓝色边框为普通场景截图
- 右侧详情面板显示配置类型（单Avatar配置 / 多Avatar组合）
- 支持为每个 Avatar 配置单独设置标签

## [0.21] - 2026.1.23

- 修复 UI 区域边框在不同 Unity 版本下显示为纯黑色的问题

## [0.22] - 2026.3.1

- 多Avatar数据状态警告：当场景从多Avatar变为单Avatar后，旧的多Avatar截图会显示红色警告图标
- Avatar缺失检测：当前加载场景中，如果 Avatar 截图引用的 Avatar 已被删除，缩略图显示警告图标
- 标题栏错误统计分离：场景文件丢失与 Avatar 数据异常分别独立计数和显示

## [0.23] - 2026.3.20

- 插件名称从 `AvatarSceneBrowser` 更名为 `VRCAvatarSceneBrowser`
- 菜单路径统一更新为 `Tools/CYN-lab/VRCAvatarSceneBrowser/`
- 修复截图按钮打开窗口时标题显示为 `MainWindow` 的问题
